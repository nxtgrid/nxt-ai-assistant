# Grafana MCP Server

Provides MCP tools for rendering Grafana dashboard panels as images.

## Overview

This server dynamically generates tools based on Grafana dashboard panels indexed by the RAG pipeline. Each enabled panel becomes an MCP tool that can render that panel with specified dashboard variables.

## Features

- **Dynamic tool generation**: Tools are automatically created from panel metadata
- **Variable support**: Pass dashboard variables as key-value pairs
- **Image rendering**: Returns panel visualizations as PNG images
- **Configurable panels**: Enable/disable specific panels via settings
- **Grafana render API**: Uses native Grafana rendering capabilities

## Configuration

Add these environment variables to your `.env` file:

```bash
# Grafana Configuration
GRAFANA_URL=https://grafana.example.com
GRAFANA_USERNAME=your-grafana-user
GRAFANA_PASSWORD=your_secure_password
GRAFANA_FOLDER_NAME=My Dashboards
GRAFANA_PANEL_DESCRIPTION_PROMPT=You are a system that generates tool descriptions...
GRAFANA_ENABLED_PANELS=dashboard_uid:1,dashboard_uid:2,dashboard_uid:5
GRAFANA_PANELS_METADATA={"dashboard_uid:1": {...}, ...}
GRAFANA_ACTIONS_ENABLED=false
```

### Configuration Details

- **GRAFANA_URL**: Grafana instance URL
- **GRAFANA_USERNAME**: Grafana username for authentication
- **GRAFANA_PASSWORD**: Grafana password for authentication
- **GRAFANA_FOLDER_NAME**: Folder name to index (used by RAG pipeline)
- **GRAFANA_PANEL_DESCRIPTION_PROMPT**: System instructions for Gemini LLM
- **GRAFANA_ENABLED_PANELS**: Comma-separated list of panel IDs to enable (format: `dashboard_uid:panel_id`)
- **GRAFANA_PANELS_METADATA**: JSON string containing panel metadata (auto-generated by indexer)
- **GRAFANA_ACTIONS_ENABLED**: Master switch to enable/disable Grafana tools

## How It Works

### 1. Panel Indexing (Nightly)

The RAG pipeline indexer (`rag_pipeline/ingestion/grafana_indexer_v2.py`):
1. Connects to Grafana API
2. Fetches all dashboards in configured folder
3. Extracts panel information (title, description, query, variables)
4. Sends to Gemini LLM to generate tool descriptions
5. Stores metadata in `GRAFANA_PANELS_METADATA` environment variable

### 2. MCP Server Initialization

The MCP server:
1. Loads `GRAFANA_PANELS_METADATA` from environment
2. Filters by `GRAFANA_ENABLED_PANELS`
3. Dynamically creates tools for each enabled panel
4. Each tool accepts variables and rendering options

### 3. Panel Rendering

When a tool is called:
1. Receives panel ID and variables as arguments
2. Builds Grafana render API URL: `/render/d-solo/{dashboard_uid}/{dashboard_name}?panelId={id}&var-{key}={value}`
3. Fetches rendered panel as PNG image
4. Returns base64-encoded image with success status

## Tool Schema

Each generated tool has this structure:

**Tool Name**: `render_{dashboard_uid}_{panel_id}`

**Input Schema**:
```json
{
  "variables": {
    "type": "object",
    "description": "Dashboard variables (e.g., grid_id, time_range)",
    "additionalProperties": {"type": "string"}
  },
  "width": {
    "type": "integer",
    "description": "Image width in pixels",
    "default": 1000
  },
  "height": {
    "type": "integer",
    "description": "Image height in pixels",
    "default": 500
  }
}
```

**Output**:
- PNG image (base64 encoded)
- JSON with success status and panel metadata

## Example Usage

### Via MCP Client

```python
# Call tool to render panel
result = await call_tool(
    server="grafana",
    tool="render_xyz789_3",
    arguments={
        "variables": {
            "grid_id": "grid_001",
            "time_range": "now-24h"
        },
        "width": 1200,
        "height": 600
    }
)

# Result contains:
# - Image content (PNG)
# - Success status
# - Panel metadata
```

### Via Admin UI

1. Navigate to Settings → Grafana Dashboard Panels
2. Set folder name (e.g., "My Dashboards")
3. Configure panel description prompt
4. Select panels to enable
5. Save settings
6. Wait for nightly sync or trigger manually

## Panel Metadata Structure

The `GRAFANA_PANELS_METADATA` environment variable contains:

```json
{
  "dashboard_uid:panel_id": {
    "title": "Power Usage",
    "description": "Shows power consumption over time",
    "dashboard_id": 123,
    "dashboard_uid": "xyz789",
    "dashboard_name": "grid-monitoring",
    "dashboard_title": "Grid Monitoring Dashboard",
    "tool_description": "Use this to view power usage. Requires: grid_id, time_range",
    "variables": ["grid_id", "time_range"],
    "panel_type": "graph"
  }
}
```

## Grafana Render API

This server uses Grafana's native render API:

**Endpoint**: `/render/d-solo/{dashboard_uid}/{dashboard_name}`

**Query Parameters**:
- `panelId`: Panel ID to render
- `var-{variable_name}`: Dashboard variable values
- `width`: Image width in pixels
- `height`: Image height in pixels
- `from`: Time range start (optional)
- `to`: Time range end (optional)

**Authentication**: HTTP Basic Auth (username/password)

**Response**: PNG image (binary data)

## Scheduling

The Grafana indexer runs nightly via:
- **APScheduler** in `chat_orchestrator/orchestrator/api/app.py`
- **Default time**: 2 AM UTC (configurable via `GRAFANA_SYNC_HOUR`)
- **Process**: Fetches panels → Generates descriptions → Updates env var

## Troubleshooting

### "Grafana actions are disabled"
Enable via Settings → Grafana Dashboard Panels → Enable Grafana Actions

### "Panel not found"
Check that the panel is in `GRAFANA_ENABLED_PANELS` and `GRAFANA_PANELS_METADATA`

### Authentication errors
Verify `GRAFANA_USERNAME` and `GRAFANA_PASSWORD` are correct

### Empty metadata
Run the indexer manually:
```bash
cd rag_pipeline/ingestion
python grafana_indexer_v2.py --folder-name "My Dashboards"
```

### Render API errors
- Ensure Grafana instance is accessible
- Check dashboard UID and panel ID are correct
- Verify variables match dashboard configuration

## Security Notes

- Credentials stored in environment variables
- HTTP Basic Auth for Grafana API
- Panel access controlled via `GRAFANA_ENABLED_PANELS`
- Images rendered server-side (not cached)
